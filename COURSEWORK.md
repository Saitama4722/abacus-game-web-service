# Текстовые блоки для курсовой работы

Ниже приведены готовые формулировки для вставки в пояснительную записку или разделы курсовой работы по теме «Веб-сервис для создания и проведения математических игр».

---

## Цель проекта

Разработка веб-сервиса для создания и проведения математических игр, позволяющего администраторам создавать игры с настраиваемыми темами и заданиями, формировать команды участников и проводить турниры с прозрачной системой подсчёта очков. В рамках проекта реализована игра «Абака» на математическом квадрате 6×6 с фиксированными баллами за задания (10–60), бонусами за полные строки и столбцы и одной попыткой на задание.

---

## Архитектура системы

Приложение построено по классической трёхуровневой схеме: уровень представления (HTML-шаблоны Jinja2, Bootstrap 5), уровень бизнес-логики (маршруты FastAPI, сервис игровой логики), уровень данных (SQLite, SQLAlchemy ORM). Обработка запросов — по схеме MVC: маршруты принимают запросы, обращаются к моделям и сервисам, передают данные в шаблоны и возвращают HTML или редиректы. Сессии хранятся на сервере (SessionMiddleware), аутентификация — по логину и паролю с хешированием bcrypt.

---

## Технологический стек

- **Бэкенд:** FastAPI — асинхронный веб-фреймворк, маршрутизация, зависимости, обработка форм и ошибок.
- **База данных:** SQLite — файловая СУБД; SQLAlchemy 2.x — ORM, декларативные модели, сессии, миграции структуры (добавление столбцов).
- **Шаблоны:** Jinja2 — серверный рендеринг HTML, наследование (base.html), блоки, фильтры.
- **Фронтенд:** Bootstrap 5 — адаптивная вёрстка, компоненты (таблицы, формы, модальные окна, навигация).
- **Сервер:** Uvicorn — ASGI-сервер для запуска приложения.
- **Безопасность:** bcrypt — хеширование паролей; CSRF-токены в формах; проверка прав по ролям (admin, moderator, player).

---

## Схема базы данных

- **users** — пользователи (логин, хеш пароля, роль, привязка к команде team_id). Роли: admin, moderator, player.
- **teams** — команды (название). Связь с users по team_id.
- **games** — игры (название, описание, флаг is_active, статус in_progress/finished, количество команд-получателей супербонуса).
- **game_teams** — связь многие-ко-многим между играми и командами.
- **topics** — темы игры (до 6 на игру, order_index 1–6).
- **tasks** — задания (по 6 на тему: текст, правильный ответ, order_index, баллы 10–60).
- **results** — результат команды в игре (одна запись на пару game–team): базовые баллы, горизонтальный и вертикальный бонусы, множитель супербонуса, итоговый счёт.
- **answers** — ответы (связь result–task, введённый ответ, признак правильности, начисленные базовые баллы).

Схема поддерживает каскадное удаление (игра → темы → задания; игра → game_teams; result → answers) и уникальные ограничения (одна запись result на пару game–team, один answer на пару result–task).

## Игровая логика (Абака)

- **Сетка:** 6 тем × 6 заданий (36 ячеек). В каждой теме задания имеют порядковые номера 1–6 и баллы 10, 20, 30, 40, 50, 60 соответственно.
- **Порядок решения:** в рамках одной темы задания решаются строго по порядку: следующее открывается только после правильного решения предыдущего.
- **Одна попытка:** на каждое задание допускается одна отправка ответа; повторная отправка для той же ячейки недоступна.
- **Баллы:** за правильный ответ начисляются базовые баллы (10–60 по позиции). Неправильный ответ — 0 баллов, ячейка считается использованной.
- **Бонус за полную строку:** если команда решила все 6 заданий одной темы (горизонталь), начисляется бонус +50 очков. Первые N команд (N задаётся в игре) получают супербонус +100.
- **Бонус за полный столбец:** если команда решила по одному заданию с одинаковым порядковым номером (1–6) во всех 6 темах (вертикаль), бонус равен стоимости задания в этой позиции (10, 20, …, 60). Первые N команд получают удвоенный бонус.
- **Итоговый счёт:** сумма базовых баллов, горизонтальных и вертикальных бонусов (с учётом супербонуса). Игра может быть переведена в статус «завершена», после чего приём ответов прекращается.

## Роли пользователей

- **Администратор (admin):** полный доступ — создание, редактирование и удаление игр и команд; настройка тем и заданий; привязка команд к играм; завершение игры; просмотр результатов. Доступ к панели администратора (/admin/).
- **Модератор (moderator):** создание и редактирование игр и команд (без удаления); настройка тем и заданий; проведение игры; возможность выбирать команду при просмотре поля и отправке ответа от её имени; завершение игры и просмотр результатов.
- **Команда (player):** пользователь привязан к команде (team_id). Может участвовать только в тех играх, в которые добавлена его команда: просмотр поля 6×6, открытие доступных заданий, отправка одного ответа на задание, просмотр результатов игры. Не имеет доступа к созданию и редактированию игр и команд.

Проверка прав выполняется в зависимостях FastAPI (require_admin, require_admin_or_moderator, require_game_access) и при отображении интерфейса (кнопки создания/редактирования/удаления по ролям).
